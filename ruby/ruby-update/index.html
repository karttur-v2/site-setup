    Gem updates

    Regardless if you used a Ruby version manager or a bare-bone installation (see previous document), you should keep your Ruby environment and its gems updated when you start a new project.

    The way to keep track of and update gems varies dependent on your Ruby installation route. You can install or upgrade gems individually, or apply a gem manager to update all gems in one go.

    At the time of installing chruby and Ruby in March 2025 on macOS, I had trouble with installing gems. I had to rebuild my setup several times. There are bugs in some combinations of Ruby versus gem versions. This apparently changed frequently as the system develop and new versions are released. It is thus not possible to give any detailed instructions, and the following suggestions are only indicative ideas for how to solve errors with versions.

    Individual gem updates

    The default command for updating a Ruby gem is to run the terminal command

    gem install <gem>

    This command, however, is primarily meant for the system default Ruby installation and not the Homebrew installation. Some Ruby managers (e.g. chruby), however, capture the above command and will install the gem with the linked chruby version of Ruby.

    For Homebrew installed Ruby, the command for installing individual gems is instead brew-gem. For using that you have to first install brew-gem:

    brew install brew-gem

    Then use the Terminal to install individual gems via Homebrew:

    brew-gem install <gem> --homebrew-ruby

    You can also upgrade gems via Homebrew:

    brew-gem upgrade <gem> --homebrew-ruby

    For more information see the page Managing Ruby Gems using Homebrew.

    Update using gem manager

    This section was largely taken from the post Update Gems by Daniel Kehoe.

    Check if your system comes with a gem manager by issuing the terminal command:

    gem -v

    If your command returns a version number, go ahead and update gem

    gem update --system

    Use the gem manager to list your gems:

    gem list

*** LOCAL GEMS ***

...

    If you later want to make sure that your gems were updates, save the listed gems.

    To check which gems are outdated:

    gem outdated
...

    You can also save this list for later if you want to control the result of the updating.

    I had some outdated gems, and updated all of them in one go:

    gem update
Updating installed gems
...

    If you saved the list of gems prior to the update, you can now rerun the gem list command and check out that the outdated gems were updated.
