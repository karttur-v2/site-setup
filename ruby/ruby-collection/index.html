    Ruby collection

    This introduction on how to install and setup Ruby for Jekyll is written as a Jekyll Collection. In reality that only means that each document is saved inside a folder named _ruby, and that I defined a collection called ruby in the _config.yml file:

    collections:
  ruby:
    order:
      - ruby-collection.markdown
      - ruby-versions.markdown
      - ruby-installation.markdown
      - ruby-update.markdown
    output: true
    permalink: /websetup/jekyll/ruby/

    As seen from the collections example above, you can set the order of the documents in the collection. To publish a collection in your _site folder, you must set output to true. Setting permalink is optional.

    For this collection on Ruby I choose the layout to be single, I defined the permalink and set my author_profile to true. The front matter of this document thus looks like this:

    ---
title:  "Ruby versions"
layout: single
permalink: /websetup/ruby/ruby-versions
author_profile: true
---
